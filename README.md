# Book Tracker: Приложение для учёта прочитанных книг

**Автор:** Лисовский Николай Андреевич
**Язык:** Python 3.x

## Описание программы

«Book Tracker» — это простое десктопное приложение с графическим интерфейсом (*GUI*), предназначенное для ведения списка прочитанных книг.
Программа позволяет добавлять новые книги с указанием названия (*Title*), автора (*Author*), жанра (*Genre*) и количества страниц (*Pages*).
Все данные сохраняются в локальный файл **`data/books.json`**, что позволяет не терять информацию при закрытии программы.
Ключевые функции включают фильтрацию списка по жанру и объёму книги.
Проект управляется системой контроля версий *Git*.
> **Примечание:** Библиотека **ttkthemes** используется для красивого оформления интерфейса (темы), но **не является обязательной**. Если она не установлена или не импортирована (*ImportError*), приложение запустится со стандартным стилем *Tkinter*.

## Инструкция по запуску

> **Для запуска требуется установленный Python версии 3.8 или выше.**
> Рекомендуется использовать **виртуальное окружение** (*venv*) для изоляции зависимостей проекта.

### 1. Клонирование репозитория

Откройте терминал или командную строку и выполните команду:
bashgit clone https://github.com/ВАШ_НИК/BookTracker.gitcd BookTracker/

### 2. Создание и активация виртуального окружения

**Для Windows:**
bashpython -m venv venv   # Создаем виртуальное окружение в папке venvvenv\Scripts\activate # Активируем его (приглашение командной строки изменится на (venv))

**Для Linux / macOS:**
bashpython3 -m venv venv   # Создаем виртуальное окружение в папке venvsource venv/bin/activate # Активируем его (приглашение терминала изменится на (venv))

### 3. Установка зависимостей

Установите библиотеки из файла `requirements.txt`: bashpip install -r requirements.txt

*(Примечание: *Tkinter* входит в стандартную библиотеку Python)*
### 4. Запуск приложения

Убедитесь, что виртуальное окружение активировано (см. шаг 2), затем выполните команду из корня проекта:
bashpython main.py

После этого откроется главное окно программы.
Если библиотека *ttkthemes* установлена и импортирована успешно — будет применена тема оформления.
Если нет — приложение запустится со стандартным серым интерфейсом *Tkinter*.

## Запуск тестов

Проект содержит модульные тесты для проверки логики работы с данными.
Для их запуска убедитесь что виртуальное окружение активировано и выполните команду из корня проекта:
bashpython -m unittest discover tests/

Вы должны увидеть вывод вида `OK`, что означает успешное прохождение всех тестов.
> *Тесты используют отдельную папку `tests/data_test`, чтобы не изменять основной файл данных приложения.*
