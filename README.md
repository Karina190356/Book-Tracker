# Book Tracker: Приложение для учёта прочитанных книг

**Автор:** Лисовский Николай Андреевич
**Язык:** Python 3.x

## Описание программы

«Book Tracker» — это простое десктопное приложение с графическим интерфейсом (GUI), предназначенное для ведения списка прочитанных книг.
Программа позволяет добавлять новые книги с указанием названия (*Title*), автора (*Author*), жанра (*Genre*) и количества страниц (*Pages*).
Все данные сохраняются в локальный файл **`data/books.json`**, что позволяет не терять информацию при закрытии программы.
Ключевые функции включают фильтрацию списка по жанру и объёму книги.
Проект управляется системой контроля версий Git.

## Инструкция по запуску

> **Для запуска требуется установленный Python версии 3.8 или выше.**
> Рекомендуется использовать **виртуальное окружение** (*venv*) для изоляции зависимостей проекта.

### 1. Клонирование репозитория

Откройте терминал или командную строку и выполните команду:
bashgit clone https://github.com/ВАШ_НИК/BookTracker.gitcd BookTracker/

### 2. Создание и активация виртуального окружения

**Для Windows:**
bashpython -m venv venv   # Создаем виртуальное окружение в папке venvvenv\Scripts\activate # Активируем его (приглашение командной строки изменится)

**Для Linux / macOS:**
bashpython3 -m venv venv   # Создаем виртуальное окружение в папке venvsource venv/bin/activate # Активируем его (приглашение терминала изменится)

*(Приглашение командной строки обычно меняется на `(venv) C:\...>` или `(venv) user@host ~/...`)*
### 3. Установка зависимостей

Установите библиотеки из файла `requirements.txt`:
bashpip install -r requirements.txt

*(Примечание: Tkinter входит в стандартную библиотеку Python)*
### 4. Запуск приложения

Убедитесь, что виртуальное окружение активировано (см. шаг 2), затем выполните команду из корня проекта:
bashpython main.py

После этого откроется главное окно программы.

## Примеры использования и тесты

Ниже приведены примеры сценариев использования приложения для проверки его функциональности.

### Сценарий 1: Добавление новой книги и проверка сохранения данных

1.  Запустите приложение (`python main.py`).
2.  Введите данные в поля формы:
    *   Название: `Мастер и Маргарита`
    *   Автор: `Михаил Булгаков`
    *   Жанр: `Роман`
    *   Страниц: `496`
3.  Нажмите кнопку **«Добавить книгу»**.
4.  **Ожидаемый результат:** Появится окно с сообщением «Книга добавлена!». Книга должна появиться в таблице.
5.  Закройте программу.
6.  Запустите программу снова (`python main.py`).
7.  **Ожидаемый результат:** Книга «Мастер и Маргарита» должна снова присутствовать в таблице. Это подтверждает корректную работу сохранения в JSON.
8.  *Проверка:* Откройте файл **`data/books.json`**. Он должен содержать объект с введенными данными в формате JSON.
9.  *Проверка:* Убедитесь, что поля ввода очищены после успешного добавления книги.
10. *Проверка:* Попробуйте удалить строку из `books.json` вручную и запустить программу. Программа не должна падать с ошибкой при чтении файла.

### Сценарий 2: Применение фильтра по жанру (без учета регистра букв — «регистр» не важен)

1.  Убедитесь, что в списке есть хотя бы одна книга жанра «Фантастика» или «Детектив».
2.  В поле фильтра по жанру введите слово `фант` или `ФАНТ`.
3.  Нажмите кнопку **«Применить фильтр»**.
4.  **Ожидаемый результат:** В таблице должны остаться только те книги, жанр которых содержит подстроку «фант» (например: «Фантастика», «Фэнтези»). Поиск должен работать независимо от того, заглавными или строчными буквами введено слово.
5.  Нажмите кнопку **«Сбросить»**.
6.  **Ожидаемый результат:** Таблица должна снова отобразить полный список всех книг.
7.  *Проверка:* Убедитесь, что фильтр работает без учета регистра букв.
